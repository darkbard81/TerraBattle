# DECISIONS

프로젝트 전역 결정과 그 이유를 기록한다.
세부 구현 판단은 각 작업 폴더 문서에서 다룬다.

## 2026-04-07

### Adopt PixiJS + React HUD
- 게임 월드 렌더링은 PixiJS를 사용한다.
- HUD와 앱 UI는 React로 분리한다.
- 이유: 2D 타일 기반 퍼즐에 필요한 렌더링 유연성을 유지하면서, 다양한 해상도 대응 UI를 쉽게 관리하기 위해.

### Document-First Workflow
- 구현 전에 해당 작업 폴더 문서를 먼저 작성하거나 갱신한다.
- 이유: 규칙 중심 프로젝트에서 구현보다 문서 계약이 먼저 고정되어야 하기 때문.

### Localize Detail Docs
- 작업 상세 문서는 루트가 아니라 각 작업 폴더에 둔다.
- 이유: 문서와 구현 컨텍스트를 가깝게 유지하고 전역 문서 비대화를 막기 위해.

### Markdown as Default
- 기본 문서 형식은 `md`다.
- `mermaid`는 각 `md` 안에 포함한다.
- 구조 검증이 필요한 경우에만 `schema.json`을 둔다.
- 이유: 문서 작성과 추출 흐름을 단순하게 유지하기 위해.
