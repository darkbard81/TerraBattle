# AGENTS

이 저장소에서 작업하는 에이전트는 이 문서를 전역 규칙으로 따른다.

## Priority

1. 사용자 직접 지시
2. 현재 작업 폴더의 문서
3. 루트 전역 문서
4. 코드 관례와 기존 구조

상위 기준과 하위 기준이 충돌하면 상위 기준을 따른다.

## Required Workflow

- 작업 전에 대상 폴더의 `md` 문서를 먼저 읽는다.
- 필요한 문서가 없으면 구현 전에 먼저 문서를 만든다.
- 문서를 바꿔야 할 경우 코드보다 문서를 먼저 수정한다.
- 구조 검증이 꼭 필요한 경우에만 같은 폴더에 `schema.json`을 둔다.
- `mermaid`는 별도 파일로 두지 말고 해당 `md` 안에 포함한다.

## Scope Control

- 요청받은 범위만 작업한다.
- 요청되지 않은 구조 확장, 기능 추가, 전역 리팩터링은 하지 않는다.
- 전역 결정이 필요한 변경은 `DECISIONS.md`에 기록 대상인지 먼저 확인한다.

## Implementation Rules

- 게임 규칙 로직과 렌더링 로직을 섞지 않는다.
- PixiJS는 월드 렌더링 중심으로 사용한다.
- React는 HUD와 앱 UI 중심으로 사용한다.
- 상태의 진실은 도메인/게임 상태에 두고 표현 계층은 이를 반영만 한다.

## Task Output Expectations

작업 완료 시 아래 내용을 남긴다.

- 무엇을 변경했는지
- 어떤 문서를 기준으로 작업했는지
- 문서와 구현 사이의 차이가 있었는지
- 남아 있는 리스크나 후속 필요 사항

## Do Not

- 문서 없이 임의로 규칙을 확정하지 않는다.
- 작업 폴더 문서를 무시하고 코드만 보고 해석하지 않는다.
- 요청되지 않은 제안을 과도하게 확장하지 않는다.
